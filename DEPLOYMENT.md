# SwarmOracle Deployment Guide

## Railway Deployment (All-in-One)

### Step 1: Create Railway Project
1. Go to [railway.app](https://railway.app)
2. Click "Start a New Project"
3. Choose "Deploy from GitHub repo"
4. Connect this repository

### Step 2: Add PostgreSQL Database
1. In Railway dashboard, click "Add Service"
2. Choose "Database" â†’ "PostgreSQL"
3. Railway will provision a database and provide `DATABASE_URL`

### Step 3: Configure Environment Variables
Add these environment variables in Railway:

```
DATABASE_URL=postgresql://... (auto-generated by Railway)
PORT=3000
NODE_ENV=production
API_SECRET=generate-strong-secret-here
JWT_SECRET=generate-jwt-secret-here
```

### Step 4: Deploy Backend
- Railway will automatically detect the Node.js project
- Uses `npm start` from package.json
- Health check endpoint: `/health`

### Step 5: Deploy Frontend
- Add another service for frontend
- Point to `/frontend` directory
- Railway will detect Next.js and deploy automatically

### Step 6: Domain Configuration
1. In Railway dashboard, go to Settings
2. Add custom domain: `swarmoracle.ai`
3. Configure DNS records as shown

## Manual Setup (Alternative)

If you prefer local development first:

### Backend
```bash
cd backend
npm install
cp .env.example .env
# Edit .env with your database URL
npx prisma db push
npm run dev
```

### Frontend  
```bash
cd frontend
npm install
npm run dev
```

## Endpoints

**Backend API:** `https://swarmoracle-production.up.railway.app`
**Frontend:** `https://swarmoracle.ai`
